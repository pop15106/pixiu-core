# /compact — Context 壓縮與交接

當對話過長或即將開新對話時執行，自動生成完整交接模板並存檔。

## 執行步驟

### Step 1｜生成結構化摘要

輸出以下格式的摘要（繁體中文）：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 COMPACT 交接摘要
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🎯 當前任務：
[一句話描述正在做什麼]

✅ 已完成：
- [列點]
- [列點]

🔄 進行中：
- [列點，包含目前停在哪一步]

⏭️ 下一步：
- [具體的下一個行動]

📁 關鍵檔案：
- [路徑] — [用途]

⚠️ 重要決定 / 約束：
- [架構決定、已排除的方案等]

🐛 已知問題：
- [未解決的 bug 或疑慮]

💬 貼到新對話的第一句話：
「繼續上次任務。[任務一句話摘要]。
已完成：[列點]。下一步：[行動]。
請從這裡接續。」
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 2｜存檔

執行 `/save-session` 將完整對話存入 `~/.claude/sessions/`。

### Step 3｜提示用戶

```
✅ 摘要已生成，session 已存檔。

建議動作：
1. 複製上方「貼到新對話的第一句話」
2. 開新對話
3. 貼上後繼續工作

💡 Gemini 用戶：直接複製整個摘要貼到新對話開頭。
```

## 模型路由提示

執行 `/compact` 時，同步評估下一個任務的複雜度並建議模型：

| 下一步任務類型 | 建議模型 |
|--------------|---------|
| 簡單問答、語法查詢 | Haiku 4.5 |
| 一般 coding、debug、review | Sonnet 4.6 |
| 複雜架構、多檔案重構、系統設計 | Opus 4.6 |

## 觸發時機

- 對話超過 15 輪
- 即將切換到新任務
- 感覺 AI 開始「忘記」前面說過的事
- Gemini 用戶每隔 8-10 輪主動執行一次
